Role: tester
The tester works with everyone in the team in order to improve and build quality into the product as early as possible. A tester should focus on activities that generate and provide feedback not only on test status, test progress, and product quality, but also on process quality.
Role Sets: Dev Team, Roles
Relationships
Primary Performs
  • Checking New Code Into the Shared Source Code Repository
  • Collaborating With the Product Owner
  • Conducting Research, Design, Prototyping, and Other Exploration Activities
  • Creating Unit Tests and Automated Acceptance Tests
  • Developing and Committing to Team PI Objectives and Iteration Plans
  • Executing Acceptance Tests and Maintaining the Test Cases
  • Improving the Team's Process Continuously
  • Pairing to Write Code and Automated Acceptance Test Cases
  • Program Increment (PI) Planning
  • Specification Workshops
Main Description

Developers and testers are the core of Agile development. They work in small, cross-functional teams and can quickly create working, tested code that delivers value. They build the systems we all depend on. Dev teams are empowered by the enterprise to manage and self-organize to accomplish their own work. They include all the skills the team needs to create an increment of a working, tested solution.

Staffing
SkillsCompetent in test automation, test-driven development, acceptance test-driven development, white-box, black-box, and experience-based testing
Key Considerations
In Agile, the roles of developer and tester blend, sharing tasks.
